# B2C Freelance Platform - Roadmap Prompts

Этот документ содержит промты для генерации всех этапов проекта фриланс-платформы с GitHub OAuth, escrow, AI Support, системой рейтингов, блокировок и апелляций. Формат подходит для отметки прогресса по каждому этапу.

---

## Этап 1. Аналитика и постановка задачи
PROMPT:
Проанализируй рынок B2C-фриланс платформ (Upwork, Fiverr, TopTal, Kwork) и подготовь подробное ТЗ для разработки закрытой платформы фриланса с GitHub OAuth, escrow через администраторские электронные кошельки, AI Support, рейтингами и системой блокировок и апелляций. Учти динамическую комиссию: 1% < 10000₽, 0.8% > 10000₽, 0.5% > 50000₽, возможность редактирования комиссии в реальном времени.

Технологии: Node.js + Express (backend), React (frontend), PostgreSQL (БД), электронные кошельки для escrow

Результат: Сравнительная таблица платформ, полное техническое ТЗ, архитектурная схема, ER-диаграмма БД

Статус: not_started

---

## Этап 2. UI/UX-дизайн
PROMPT:
Разработай детальный UI/UX-прототип для платформы с ролями Заказчик/Исполнитель. Экран регистрации через GitHub, лента задач, карточка задачи, профиль исполнителя, escrow и апелляции, панель модератора. Укажи навигацию, поведение элементов, состояния при блокировках, апелляциях и спорах.

Технологии: Figma/React для прототипа, Tailwind CSS для стилей

Результат: Описание макета Figma с компонентами, сеткой, отступами, цветовой схемой и интерактивными состояниями

Статус: not_started

---

## Этап 3. Backend: Авторизация, БД, Escrow, блокировки и апелляции
PROMPT:
Разработай backend на Node.js + Express с PostgreSQL для фриланс-платформы. Реализуй GitHub OAuth, CRUD задач и предложений, escrow через администраторские кошельки, динамическую комиссию, систему блокировок недобросовестных пользователей, апелляции, модерацию, ручное одобрение переводов при споре, блокировку по железу. Подготовь SQL DDL, OpenAPI спецификацию, описание состояния escrow (INIT, FUNDED, IN_PROGRESS, PENDING_RELEASE, RELEASED, REFUNDED, DISPUTE).

Технологии: Node.js + Express, PostgreSQL, JWT, администраторские кошельки, REST API, Redis для очередей и кэша

Результат: Полный backend-проект, база данных, спецификация API, state-machine escrow и блокировки, готовность к интеграции фронтенда

Статус: not_started

## Этап 4. Frontend: Лента, профили, карточки задач
PROMPT:
Разработай React SPA для платформы с использованием Tailwind CSS. Реализуй вход через GitHub, ленту задач с фильтрацией, карточку задачи, отправку приватного предложения, профиль исполнителя, панель заказчика, уведомления о блокировках, апелляциях и спорных сделках. Используй глобальное состояние через Zustand или Redux Toolkit, React Router для навигации.

Технологии: React + TypeScript, Tailwind CSS, Zustand/Redux Toolkit, React Router

Результат: Frontend SPA с компонентами TaskCard, ProposalModal, ProfileCard, EscrowStatus, AppealForm, интеграция с моковыми API

Статус: not_started

## Этап 5. AI Support и рейтинги
PROMPT:
Разработай модуль AI Support с многоуровневой системой: FAQ → AI → оператор. Настрой порог confidence для эскалации, сбор контекста сообщений, истории задач, транзакций. Реализуй рейтинг с защитой от необоснованных оценок, заморозку рейтинга при подозрении на мошенничество, апелляции через прикрепление доказательств. Подготовь архитектуру AI-модуля и API для апелляций и ручного пересмотра рейтингов.

Технологии: Node.js + Express, LLM API (OpenAI/Local), PostgreSQL для рейтингов, REST API для апелляций

Результат: AI Support модуль, логика рейтингов и защиты, API для апелляций, готовый к интеграции с frontend и backend

Статус: not_started

Технологии: Jest, Playwright, Node.
js, PostgreSQL, Redis, Sentry, Prometheus/Grafana

Результат: Документ тестирования и безопасности с таблицами сценариев, отчёты о проверках, рекомендации по деплою

## Этап 7. Альфа-запуск и деплой
PROMPT:
Подготовь план развертывания и альфа-запуска фриланс-платформы. Используй Docker для backend, React SPA для frontend, PostgreSQL и Redis, CI/CD через GitHub Actions, хостинг на AWS EC2/S3/RDS или Render/Vercel. Настрой домен, SSL, WAF, вебхуки GitHub OAuth, электронных кошельков, процедуру запуска альфы с инвайтами и сбором обратной связи, чек-лист готовности к продакшену.

Технологии: Docker, Node.js + Express, React, PostgreSQL, Redis, GitHub Actions, AWS/Render/Vercel, CI/CD, мониторинг и логирование

Результат: Полностью развернутая альфа-версия платформы, готовая к тестированию и сбору обратной связи
